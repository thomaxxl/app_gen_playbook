# Phase 8 Summary

Use this phase for independent QA validation immediately before final delivery.

This phase controls:

- live `app/run.sh` validation
- basic user-flow testing
- explicit QA rejection of blank, crashed, metadata-heavy, or erroring builds

It does not replace product acceptance or CEO final approval.
