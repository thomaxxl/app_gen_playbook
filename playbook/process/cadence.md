# Cadence

Recommended recurring checkpoints:

## Product / architecture review

Participants:

- Product Manager
- Architect

Purpose:

- refine scope
- resolve requirement ambiguity
- approve contract-affecting changes

## Architecture / frontend / backend sync

Participants:

- Architect
- UX/UI + Frontend
- Backend

Purpose:

- catch contract drift early
- review naming and runtime assumptions
- remove handoff blockers

## Product demo review

Participants:

- Product Manager
- UX/UI + Frontend
- Backend
- Architect

Purpose:

- validate progress against user workflows
- catch usability or domain mismatches before they harden
