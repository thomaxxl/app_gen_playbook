# Frontend Core Read Set

Compatibility wrapper only.

New task bundles MUST choose the stage-specific Frontend read set instead of
using this file directly:

- `frontend-design-core.md` for Phase 3 UX/design work
- `frontend-implementation-core.md` for Phase 5 implementation work

If a legacy task still cites `frontend-core.md`, load only the one
stage-specific manifest that matches the current task. Do not preload both
unless the inbox item explicitly spans both design and implementation.
