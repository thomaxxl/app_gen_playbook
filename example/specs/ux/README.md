# UX Artifacts

This directory holds UX/UI + Frontend-owned artifacts for Phase 3.

For the starter playbook, these files contain baseline UX decisions for the
starter app. Replace or refine them when a project needs a different landing
page, navigation model, or visibility matrix.

This directory README is reference guidance for the playbook. It is not itself
a run artifact and does not participate in artifact-status workflow.

Required files:

- `navigation.md`
- `screen-inventory.md`
- `field-visibility-matrix.md`
- `custom-view-specs.md`
- `state-handling.md`

Use this directory as the persistent output of Phase 3. Do not hide UX
decisions inside `../architecture/`, inbox messages, or implementation files.

Metadata convention:

- every file in this directory starts with the standard metadata block
- set `status: ready-for-handoff` when the UX package is ready for Architect
  review or implementation
