Keep rules separate from models and transport code.

The Northwind validation backend is not a good generic LogicBank starter, so
use the small reusable starter pattern instead.

When these templates are applied to a real run, rule implementation and rule
tests SHOULD remain traceable to stable rule IDs from
`runs/current/artifacts/product/business-rules.md`.
