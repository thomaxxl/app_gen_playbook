# `app/.gitignore`

Use a root-level `.gitignore` so the generated app can be moved into its own
repository without first inventing a new ignore policy.

```gitignore
# Python
.venv/
venv/
backend/.venv/
backend/.deps/
backend/**/__pycache__/
backend/**/*.py[cod]
.pytest_cache/
.mypy_cache/
.ruff_cache/

# Frontend
frontend/node_modules/
frontend/dist/
frontend/coverage/
frontend/playwright-report/
frontend/test-results/
frontend/*.tsbuildinfo
frontend/.vite/

# Local runtime overrides
.runtime.local.env

# Local runtime data
backend/data/*.sqlite
backend/data/*.sqlite3
backend/data/*.db
backend/data/uploads/
backend/data/media/

# OS/editor noise
.DS_Store
Thumbs.db
```

Notes:

- Keep this file at the generated app root.
- The generated app MUST be able to become its own git repo without needing a
  second pass to ignore local dependency/runtime artifacts.
- If the app adds durable upload/media storage under another path, extend this
  ignore file accordingly.
