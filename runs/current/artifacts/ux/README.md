# Current Run UX Artifacts

Create run-specific UX artifacts in this directory.

Use `../../../specs/ux/` as the template source.

The run-owned UX artifacts SHOULD preserve the section structure and tables
from the generic `specs/ux/` templates instead of collapsing back into terse
prompt-like notes.

The current run's UX package MUST include `landing-strategy.md` because the
primary entry experience is a core UX concern for every run.
